Phase 3: Reorganize directory structure for better semantic clarity

This commit completes Phase 3 of the package restructuring, improving
the organization and naming of core modules for better maintainability.

## Directory Renames (Git History Preserved)

### core/modules → core/classification (29 files)
More descriptive name that clearly indicates ASPRS/BD TOPO classification
functionality rather than generic "modules".

Files moved:
- classification_thresholds.py, advanced_classification.py
- ground_truth_refinement.py, parcel_classifier.py
- building_detection.py, transport_detection.py
- config_validator.py, serialization.py, patch_extractor.py
- And 20 more classification-related modules

### features/core → features/compute (12 files)
Eliminates confusion with top-level "core" package. More accurately
describes the module's purpose (feature computation).

Files moved:
- eigenvalues.py, normals.py, curvature.py
- architectural.py, density.py, height.py
- gpu_bridge.py, geometric.py, unified.py
- And 3 more feature computation modules

## Internal Import Updates (10 files)

Updated all internal package imports to use new paths:
- core/processor.py: .modules.* → .classification.*
- features/__init__.py: .core.* → .compute.*
- features/strategy_cpu.py, gpu_processor.py, features_gpu.py
- features/features_gpu_chunked.py
- preprocessing/__init__.py
- io/wfs_ground_truth.py

## Backward Compatibility (Zero Breaking Changes)

Added compatibility modules to maintain v3.x API:
- core/__init__.py: _ModulesCompatibilityModule
- features/__init__.py: _CoreCompatibilityModule

Old import paths continue working with deprecation warnings:
```python
# OLD (deprecated but working in v3.x)
from ign_lidar.core.modules.classification_thresholds import ...
from ign_lidar.features.core.eigenvalues import ...

# NEW (recommended)
from ign_lidar.core.classification.classification_thresholds import ...
from ign_lidar.features.compute.eigenvalues import ...
```
